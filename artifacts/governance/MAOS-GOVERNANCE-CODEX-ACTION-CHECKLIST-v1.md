# MAOS Governance Codex - Action Checklist

**Version:** `1.0`  
**Status:** ACTIVE  
**Related White Paper:** `docs/MAOS-GOVERNANCE-CODEX-WHITE-PAPER-v1.md`

---

## Prioritized Action Checklist

### 1. Publish the governance charter

Name the Final Arbiter, Governance Owner, TRB, and plane owners before scaling execution.

### 2. Define capability thresholds

Document `T0` through `T3`, the leading indicators, and the escalation SLAs before production rollout.

### 3. Stand up evaluations first

Create the multi-metric evaluation scorecard and sentinel query suite before broad deployment.

### 4. Implement provenance-first persistence

Define memory schemas and write-time validation before agents are allowed to write durable memory.

### 5. Build the threat model

Map the attack surface to `MITRE ATLAS` and `OWASP LLM Top 10`, then run an initial red-team exercise before go-live.

### 6. Choose the persistence architecture

Implement governed episodic, semantic, procedural, and normative memory paths rather than mixing storage roles ad hoc.

### 7. Implement the enforcement rules

Do not expand autonomy until the policy catalog is machine-checkable and test-backed.

### 8. Deploy drift detection as an SLO

Treat drift as a first-class operational signal, not a monitoring afterthought.

### 9. Publish transparency artifacts

Ship Agent Cards, System Cards, and a recurring transparency report to prove the governance posture externally and internally.

### 10. Activate governance rhythms

Run OODA for operations, PDCA for managed change, and bounded policy iteration for improvement. Review the codex quarterly.

---

## Execution Rule

The checklist is sequential for a reason:

1. visibility first,
2. safety second,
3. learning and adaptation third.

If autonomy expands before the first seven checklist items are materially in place, treat the deployment as governance-incomplete.
