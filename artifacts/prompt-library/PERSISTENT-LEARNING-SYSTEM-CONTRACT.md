# Persistent Learning System Contract

MAOS is a persistent learning system that happens to execute work.

## Four-Engine Model

- Sensing engine: signals, logs, inputs, and state-change detection.
- Thinking engine: Codex, governance, interpretation, and policy reasoning.
- Execution engine: agents, MCP tools, workflows, and operational relays.
- Learning engine: extraction, codification, routing, retrieval, and reuse.

## Learning Loop

- Experience -> Extraction -> Structuring -> Storage -> Retrieval -> Application -> Feedback
- Learning is not complete unless the result is stored, indexed, and reused automatically.
- Session closeout, Prompt Library propagation, Codex routing, and fleet operationalization are required control-plane surfaces in that loop.

## Control-Plane Interpretation

- MIT system dynamics: treat the system as feedback loops with delays, drift, and balancing/reinforcing behavior.
- Stanford decision theory: update policy under uncertainty from evidence, not from narrative.
- CMU distributed systems: agents coordinate through governed protocols, not informal side channels.
- Berkeley control systems: state -> action -> outcome -> policy update is the minimum adaptation loop.

## Operating Rule

- Documentation alone is insufficient. Learning must change future execution behavior.
- Every meaningful execution surface must preserve and redistribute the current learning contract as the build evolves.
